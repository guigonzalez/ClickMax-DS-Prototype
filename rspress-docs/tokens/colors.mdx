# Colors

The color system uses a **3-layer token architecture** for consistent, semantic, and maintainable styling based on the ClickMax brand identity.

## Token Layers

```
┌─────────────────────────────────────────┐
│         Component Tokens                │  ← Specific by component
│    (button-bg, input-border, etc)       │
├─────────────────────────────────────────┤
│          Semantic Tokens                │  ← Purpose-based (USE THESE!)
│  (bg-primary, text-secondary, etc)      │
├─────────────────────────────────────────┤
│         Primitive Tokens                │  ← Raw values
│   (fluor-500, neutral-900, etc)         │  ← NEVER use directly!
└─────────────────────────────────────────┘
```

:::danger Never Use Primitives Directly
Always use **semantic tokens** in your components. Primitive tokens are implementation details that can change.
:::

## Usage Guidelines

### ✅ Do

**Use semantic tokens** for styling:

```tsx
// ✅ CORRECT
<div className="bg-bg-primary text-text-primary border-border-default">
  Content
</div>
```

**Group related elements** with consistent colors:

```tsx
// ✅ CORRECT
<Card className="bg-bg-secondary">
  <CardHeader className="border-b border-border-default">
    <CardTitle className="text-text-primary">Title</CardTitle>
    <CardDescription className="text-text-secondary">Description</CardDescription>
  </CardHeader>
</Card>
```

**Use feedback colors** for states:

```tsx
// ✅ CORRECT
{error && (
  <div className="bg-feedback-error-bg border-l-4 border-feedback-error p-4 rounded">
    <p className="text-feedback-error-text text-sm font-medium">
      {error}
    </p>
  </div>
)}
```

**Use brand colors for primary actions**:

```tsx
// ✅ CORRECT
<Button className="bg-interactive-default hover:bg-interactive-hover text-neutral-900">
  Primary Action
</Button>
```

### ❌ Don't

**Don't use primitive tokens directly**:

```tsx
// ❌ WRONG
<div className="bg-fluor-500 text-neutral-900 border-neutral-200">
  Content
</div>
```

**Don't hardcode color values**:

```tsx
// ❌ WRONG
<div style={{ backgroundColor: '#D4FF00', color: '#1F2123' }}>
  Content
</div>
```

**Don't mix semantic and primitive tokens**:

```tsx
// ❌ WRONG
<div className="bg-bg-primary text-neutral-900">
  Content
</div>
```

## Dark Mode

All semantic tokens automatically switch in dark mode:

```html
<!-- Light mode (default) -->
<html>
  <div className="bg-bg-primary text-text-primary">
    Light theme content
  </div>
</html>

<!-- Dark mode -->
<html data-theme="dark">
  <div className="bg-bg-primary text-text-primary">
    Dark theme content (automatically inverted)
  </div>
</html>
```

### How It Works

Tokens are defined in `src/styles/globals.css` using Tailwind CSS 4's `@theme` block:

```css
@theme {
  /* Light mode (default) */
  --color-bg-primary: #ffffff;
  --color-text-primary: #1f2123;
  --color-interactive-default: #d4ff00;
}

/* Dark mode */
[data-theme='dark'] {
  --bg-primary: #1f2123;
  --text-primary: #ffffff;
  /* Interactive colors stay the same - brand consistency */
}
```

This generates utility classes like:
- `bg-bg-primary`
- `text-text-primary`
- `border-border-default`
- `bg-interactive-default`

## Accessibility

### Contrast Ratios

All critical color combinations meet **WCAG 2.1 AA** requirements (minimum 4.5:1 for normal text, 3:1 for large text).

**High Contrast Combinations**:

| Combination | Contrast Ratio | Level | Usage |
|-------------|----------------|-------|-------|
| `neutral-900` on `neutral-0` | 21:1 | AAA | Primary text on white |
| `neutral-600` on `neutral-0` | 10.4:1 | AAA | Secondary text on white |
| `neutral-900` on `fluor-500` | 8.2:1 | AAA | Text on brand buttons |
| `torch-800` on `torch-50` | 7.1:1 | AA | Error text on error bg |
| `azure-800` on `azure-50` | 6.8:1 | AA | Success text on success bg |

**Brand Interactive Colors**:

| Combination | Contrast Ratio | Level | Usage |
|-------------|----------------|-------|-------|
| `fluor-500` on `neutral-900` | 12.5:1 | AAA | Brand accent on dark |
| `neutral-900` on `fluor-500` | 8.2:1 | AAA | Dark text on brand |
| `azure-500` on `neutral-0` | 3.4:1 | AA (large) | Secondary buttons |

:::tip Testing Contrast
Use browser DevTools or tools like [WebAIM Contrast Checker](https://webaim.org/resources/contrastchecker/) to verify custom color combinations.
:::

### Color Blindness

The color system is designed to be distinguishable for common types of color blindness:

- **Protanopia/Deuteranopia** (red-green): Uses fluorescent yellow-green (Fluor) and hot pink (Torch) instead of traditional red/green
- **Tritanopia** (blue-yellow): Feedback colors have sufficient brightness and saturation differences
- **Achromatopsia** (total): All elements work in grayscale through contrast and shape

**Best practices**:
- Don't rely on color alone to convey information
- Use icons + text + color for states
- Provide text labels for interactive elements
- Ensure sufficient contrast in all color combinations

## Color Psychology & Brand Identity

The ClickMax color system is built around the vibrant **Fluor** (fluorescent yellow-green) brand color:

- **Fluor (Yellow-Green)**: Energy, innovation, forward-thinking. Main brand identifier.
- **Azure (Bright Blue)**: Trust, reliability, success. Secondary brand color.
- **Torch (Hot Pink)**: Urgency, attention, critical actions.
- **Cadmium (Yellow)**: Caution, warning, informational alerts.
- **Purplish (Deep Purple)**: Intelligence, creativity, informational content.
- **Neutral (Grayscale)**: Balance, professionalism, content hierarchy.

## Migration Guide

If you're migrating from the old color system:

```tsx
// Before (old palettes: yellow, gray, red, orange, green, blue)
<div className="bg-gray-50 text-gray-900 border-gray-200">
  <button className="bg-yellow-500 hover:bg-yellow-600">
    Action
  </button>
</div>

// After (new palettes: neutral, fluor, azure, purplish, cadmium, torch)
<div className="bg-bg-primary text-text-primary border-border-default">
  <button className="bg-interactive-default hover:bg-interactive-hover text-neutral-900">
    Action
  </button>
</div>
```

**Key Changes**:
- `yellow-*` → `fluor-*` (brand primary color)
- `gray-*` → `neutral-*` (grayscale)
- `red-*` → `torch-*` (error/destructive)
- `orange-*` → `cadmium-*` (warning)
- `green-*` → `azure-*` (success)
- `blue-*` → `purplish-*` (info)

**Benefits**:
- ✅ Stronger brand identity with distinctive Fluor color
- ✅ Better accessibility with tested contrast ratios
- ✅ More vibrant, memorable color palette
- ✅ Improved color blindness support
- ✅ Semantic naming (fluor = brand, torch = error)
